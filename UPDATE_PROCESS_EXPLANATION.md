# الطريقة الصحيحة لتحديث المشروع مع الحفاظ على التعديلات المحلية

هذا الملف يشرح الخطوات التي اتبعناها لتحديث مشروع SpotiFLAC-Mobile إلى أحدث إصدار (v3.8.8) من المستودع الرسمي على GitHub (zarzet/SpotiFLAC-Mobile) دون فقدان أي من الميزات والتعديلات المخصصة التي تمت برمجتها محلياً.

## المشكلة
كان لدينا كود مخصص (Mini Player Styles, Now Playing Screens, Streaming Fallbacks) مكتوب محلياً وغير مرفوع (Uncommitted). في نفس الوقت، صدر تحديث جديد للمشروع الأصلي يحتوي على ميزات جديدة (مثل الإكمال التلقائي للبيانات AutoFill). الهدف كان دمج الاثنين معاً بأمان.

## الخطوات التي تم تنفيذها (باستخدام Git)

### 1. حفظ التعديلات المحلية مؤقتاً (Stash)
قمنا بتنفيذ الأمر:
```bash
git stash
```
**ماذا يفعل؟** يقوم بأخذ جميع التعديلات والملفات التي برمجتها، ويخفيها في "صندوق آمن" (Stash)، ويُعيد المشروع ليكون نظيفاً ومطابقاً للنسخة القديمة (v3.8.5) كما هي في المستودع.

### 2. جلب التحديثات الجديدة من مستودع GitHub
قمنا بتنفيذ الأمر:
```bash
git pull origin main
```
**ماذا يفعل؟** يقوم بتحميل كل الملفات، الميزات الجديدة، والإصلاحات من النسخة v3.8.8 ودمجها في المشروع النظيف.

### 3. استرجاع التعديلات المحلية ودمجها مع التحديث الجديد
قمنا بتنفيذ الأمر:
```bash
git stash pop
```
**ماذا يفعل؟** يفتح "الصندوق الآمن" ويحاول كتابة تعديلاتك المحلية المخصصة فوق الأكواد الجديدة التي تم تحميلها في الخطوة السابقة.

## ⚠️ ماذا يحدث عند وجود تعارض (Merge Conflict)؟
خلال الخطوة الثالثة، اندمجت أغلب الملفات بنجاح وفي صمت (مثل `main_shell.dart` و `linx_player` و `album_screen.dart` وغيرها). التحديث الذكي لنظام Git أدرك كيفية وضع أكوادك الجديدة والميزات الخاصة بالمشغل بجانب التعديلات الرسمية.

ولكن، ظهر **تعارض (Merge Conflict)** في ملف `track_metadata_screen.dart`.
* **السبب:** التحديث الجديد (v3.8.8) أضاف مئات الأسطر البرمجية في هذا الملف لبرمجة ميزة "الإكمال التلقائي للبيانات". وفي نفس الوقت، تعديلاتك المحلية (Stash) كانت قد عدلت بعض المسافات وتنسيق الأكواد (Formatting) في نفس الملف بالضبط. فلم يعرف برنامج Git أيهما يختار.
* **الحل الذي قمنا به:** قمنا بتحليل سياق التعارض، ووجدنا أن تعديلاتك في هذا الملف كانت مجرد تنسيقات بسيطة وليست ميزات (لأن الميزات موجودة في ملفات أخرى). بينما التحديث الجديد يحمل الميزة الضخمة. لذلك، أعطينا الأولوية لنسخة التحديث (Upstream) في هذا الملف المعين لكي لا نفقد الميزة الجديدة، وتجاهلنا التنسيقات القديمة مؤقتاً، مما أدى لمسح التعارض ودمج المشروع بنجاح.

### 4. تنظيف صندوق الحفظ
بعد حل التعارض بنجاح وتأكدنا أن كل الأكواد سليمة والأمور مستقرة، قمنا بحذف التخزين المؤقت (صندوق الـ Stash) لتنظيف بيئة العمل عبر:
```bash
git stash drop
```

## خطوات إضافية (تحديث رقم الإصدار)
لتجنب ظهور نافذة "تنبيه: يوجد تحديث جديد جاهز للتحميل" داخل التطبيق، قمنا بتحديث رقم الإصدار الداخلي يدوياً ليطابق التحديث الجديد:
1. **تعديل `lib/constants/app_info.dart`**: جعلنا الإصدار `3.8.8` بدلاً من النسخة القديمة. وأزلنا شرط `kDebugMode` لكي يظهر الرقم الحقيقي `3.8.8` في إعدادات التطبيق للمطور دائماً بدلاً من كلمة `Internal`.
2. **الحفاظ على قوة ونظافة الكود**: مسحنا المكتبات الزائدة (Unused Imports - مثل مكتبة Foundation) الناتجة عن التعديل الأول للحفاظ على نظافة واحترافية الكود.
3. **تحديث الـ `pubspec.yaml`**: ليتوافق مع أرقام الإصدارات الصحيحة ويصبح الكود جاهزاً تماماً للتشغيل.

بهذه الطريقة، أصبح المشروع الآن يجمع بين **قوة التحديث الرسمي وثباته v3.8.8** وبين **جمال الميزات والتصاميم المخصصة التي طورناها لك**.
